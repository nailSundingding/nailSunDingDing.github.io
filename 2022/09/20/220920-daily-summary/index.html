FIRST_HALF

上午修改 Fusion Design 中 Step 组件的 UI 。因为库没有提供很多的修改样式的 api ，所以智能自己打开 浏览器的 F12 去一个一个的剥开层级，修改样式。。。

这样的速度是比较慢的，而且修改起来也比较费事、费时。。。

修改样式的时候，会发现自己的 HTML, CSS, CSS3 基础是比较薄弱的。。。

准备复习一波。。

需要解决的问题

我们在使用前端UI库时，会使用到一些国内外比较有名的大型UI库，对于他们 组件的引用方式，背后的原理，以及高效的对它的UI样式进行自定义定制。。。

SECOND_HALF

1、下午编写了一个输入校验，业务场景就是一个 input ，用户输入一个版本号，如 1.1.1，后台做的限制是最高值为 99.99.99，每次会在后端查询到当前的最新版本。。版本是必须要大于前一个版本的，并且是符合格式的。

​ 没什么好说的，就是这个校验里面会涉及到 正则表达式的应用，突然发现自己对于JS中正则表达式，RegExp 类的，升值与其他的基本类型的方法，有点淡忘了，要注意复习一下。

2、Table 中 每个 Item 中的某些项，单行显示，超出单行省略显示。。

​ 这里今天闹了一个经验不足的笑话，就是自己最开始做的操作就是 查询选项字符串的长度，大于某个值就截断，通过 slice() 方法进行截取操作，然后。。后面的垃圾操作懂都懂的。。。

明显是不能这样操作的，被一个经验老道的大哥一眼看出，直接否定，老大哥封装了一个组件，然后通过将显示的值传递给组件，然后去做出这样一个效果。。。

具体怎么操作的明天再看看，以及如何的封装出这样的可复用的组件，，也是值得学习的。

需要解决的问题：

1、深入正则表达式的书写。。。

2、学习这个文字截取显示的操作。。。
