FIRST_HALF

修改两个非常小的业务需求。

遇到一个细节的问题：

使用 Fusion Design 的 Dialig 时，发现自己写的圆角样式在初始打开的时候时不生效的，要刷新一下网页，才能出现给 Dialog 设置的自定样式。

想着这个应该是自己设置的自定义样式是不生效的，所以尝试着将 className 移动到了 width 和 height 之上。

问题仍然得不到解决。。。

最后解决问题的方式，是发现，在页面从首页跳转过来的时候，dialog 的样式被覆盖掉了。

是在 Home 页面里面也了一个莫名其妙的 dialog ，这个默认的 dalog 的默认样式，把自己写的自定义样式给覆盖掉了。

需要解决的问题

这里的操作逻辑是在 Home 页面下，点击 “页面管理”，回跳转到页面管理的页面，在页面管理页面点击”添加新页面”，会弹出 Dialog，这个 Dialog 就是我们自定义修改的样式。

在逻辑操作上，是先有 Home ，后有 Dialog ,但是这里， 在 Home 的 index 文件里面的 最后写的一个 Dialog 组件，会在 “页面管理” 的 Dialog 之后加载。。。

难道 React 的父子组件渲染方式同 Vue,先渲染 并挂载子组件，后渲染挂在父组件。。。

下去研究一下。
