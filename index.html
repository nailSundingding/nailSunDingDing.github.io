（2）在父组件触发 特定子组件的方法

回想一下上面的问题，在 子 组件中点击更新按钮，子组件将会调用方法，并依次去触发 父 、爷 组件的方法，最后在 爷 组件去发送数据给后台，在后台返回一个 是否成功的标志之后，再将这个标志从 爷 依次传递给 父、子。最后子再根据这个 是否成功的标志位，去做出相应的变化。

第一次尝试，是在 爷 组件直接设置一个 updateSucceed 标志位（true / false )，代表成功与否，然后再将这个标志位分发到 父 ，再由 父 传递给 子 。再爷 这里有个操作：

能看到 爷 是根据一个数组去 map 出很多的 父 ，所以 updateSucceed 会传递给所有的父组件，，，

那么这里再，某个特定的 子 组件中触发 更新事件，进而触发特定的父组件发生变化，进而触发 爷 发生更新标志位，像上面 这样的 updateSucceed={updateSucceed}，如果直接去更改爷 组件一级的状态，会导致所有的 父 组件的标志位发生改变，而出现，点击某一个 子 组件，获得后台返回的状态后，所有的 子 组件都发生了变化。

现在放弃上面的思路，不直接去改变爷 组件中的updateSucceed 标志位，而是在爷 组件收到后台响应后，去触发对应的父 组件状态变化。

那么 爷 怎么去单独触发 父 组件中的状态变化呢？

这里想到的是 useRef()，使用 爷 组件去调用 子 组件中的方法，从而单独改变相应状态。

(https://www.jb51.net/article/260197.htm)

这样可以单独触发 特定的 父 组件中的事件，来单独的改变状态。

但是还有问题。。。

这个是会发现，不论你点击哪个子组件的更新按钮，爷 组件都只会去调用最后一个 父 组件的setUpdateSucceed 方法。

也就是在父 组件中使用 ref={ChildRef} ，map 下来之后，绑定的是最后的那个 父 组件。

(3) 子 触发事件时，将特定的父 组件绑定给 爷 组件的 childRef

现在出现问题是 爷 使用了 childRef 以后，map 循环绑定的时最后一个 父 组件，，，

现阶段想到的解决办法是，在 子 组件触发了 点击事件以后，会传递更新操作，会一层层的函数回调，父 组件会将点击的组件 id 与 latestServiceId 传递上来， 给到爷 组件。 传递上来时，在 爷 组件设置一个参数接收 这个 id，然后在爷 组件的 jsx 中给 父 组件的 ref 绑上 childRef 值。

可以说是自己的问题，因为在分页处理的时候，取到后端的数据，将后端的数据筛出totalRows和pageSize，然后将这两个值赋给 分页组件 <Pagination />，去显示返回的总数据和当前页条数。因为后端返回了 0 和 0，这时，不知道为什么UI库会出现问题，会给分页数据赋值一个NaN。
